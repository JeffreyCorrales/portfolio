Platform:
Web Browser

Django,

Overview


Project Hex was a way to assist content creators in functioning as quest givers. It consisted of two parts: a front end client and a back end content management system (CMS).

The front end client was a fake computer terminal that ran in your browser. A quest reciever would be prompted to enter a hexadecimal access token. The token would then be authenticated and the user would gain access to various types of content such as:

  • Hacking into a Fleeca bank account and transfering funds to a stateID
  • Hacking into a Life Invader email account and extracting a GPS Map with marked coordinates.
  • Engaging back and forth in conversation with a security AI Agent who challenges you to succeed in a simulated high risk scenario
  • Browsing dark web intel gathered by a mysterious entity and paying ButC to have the compromising blackmail purged

The back end was a CMS that allowed quest givers to login and create and manage fake bank accounts, access tokens, AI agent prompts, map markers and more in real time. The web based user interface was easy to use and lightweight.
